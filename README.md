# MacBook Air CPU sustained-load test

这个目录里的 `cpu_throttle_test.py` 用来测试 MacBook Air 在 CPU 长时间满载时的性能曲线。它会启动多个 Python 进程持续做 SHA256 计算，并按固定时间窗口输出吞吐量。

重点看的是：随着时间推移，同样长度的时间窗口里还能完成多少计算。如果吞吐量持续下降，通常就说明机器开始进入热限制或降频区间。

## 快速开始

直接运行默认测试：

```bash
python3 cpu_throttle_test.py
```

默认参数：

```text
测试时长: 45 分钟
采样窗口: 10 秒
预热时间: 5 秒
worker 数量: 当前机器逻辑 CPU 数
baseline: 前 3 个采样窗口的平均吞吐量
```

推荐插电运行，并关闭低电量模式。测试时尽量不要同时跑其他重负载任务。

## 输出含义

示例：

```text
Workers: 10 | Bucket: 10s | Warm-up: 5s | Duration: 45:00 | Baseline buckets: 3
Starting workers...
Warming up for 5s...

   #  Elapsed       Hashes       Rate/s   Relative   Status   OK
----------------------------------------------------------------------------
   1    00:10       28,420      2,842.0      99.8%     peak  yes
   2    00:20       28,510      2,851.0     100.1%     peak  yes
   3    00:30       28,550      2,855.0     100.2%     peak  yes
   4    00:40       27,100      2,710.0      95.1%     peak  yes
   5    00:50       25,600      2,560.0      89.8%     drop  yes
```

字段说明：

```text
#          第几个采样窗口
Elapsed    正式测试开始后的已运行时间，不包含 warm-up
Hashes     这个窗口里完成的 SHA256 次数
Rate/s     每秒完成的 SHA256 次数
Relative   相对 baseline 的性能百分比
Status     peak / watch / drop
OK         是否收齐了所有 worker 的该窗口数据
```

`Status` 的判断规则：

```text
peak   >= 95%
watch  >= 90% 且 < 95%
drop   < 90%
```

如果连续多个窗口都是 `drop`，就可以认为性能已经明显下降。默认 10 秒一个窗口，可以用“连续 6 个 drop”作为 1 分钟级别的稳定降频判断。

## 为什么有 warm-up

脚本默认有 5 秒 warm-up：

```text
先启动所有 worker
让它们进入 CPU 满载循环
5 秒后才开始正式计数
```

这样第一条正式数据不会混入 Python 进程创建、解释器初始化、第一次调度等成本。这个 warm-up 很短，目的不是把机器预热，而是排除启动噪声。

## 常用命令

跑 30 分钟：

```bash
python3 cpu_throttle_test.py --duration 30
```

5 秒一个采样窗口，更细地看早期变化：

```bash
python3 cpu_throttle_test.py --bucket 5
```

只预热 3 秒：

```bash
python3 cpu_throttle_test.py --warmup 3
```

指定 worker 数量，例如 8 个进程：

```bash
python3 cpu_throttle_test.py --workers 8
```

把结果保存成 CSV：

```bash
python3 cpu_throttle_test.py --csv results/mba_cpu_test.csv
```

更敏感地使用第 1 个窗口作为 baseline：

```bash
python3 cpu_throttle_test.py --baseline-buckets 1
```

## 同时记录 macOS 热状态

可以另开一个终端运行：

```bash
sudo powermetrics --samplers cpu_power,thermal -i 5000 -o ~/Desktop/mba_thermal.log
```

然后再运行测试脚本。测试结束后在 `powermetrics` 终端按 `Control + C` 停止记录。

`powermetrics` 可以辅助观察 CPU 功耗和 thermal pressure；但判断“什么时候开始降频”，建议优先看脚本输出里的 `Relative` 和 `Status`。

## 怎么判断开始降频

建议使用这个规则：

```text
第一次连续 6 个 10 秒窗口低于 90%，即连续约 1 分钟 Status=drop
```

这比单个窗口掉到 90% 以下更可靠，因为单个窗口可能会受系统调度影响。

最终可以记录两个结论：

```text
开始明显下降: 第几分钟附近首次连续 drop
稳态性能: 第 20-30 分钟 Relative 的平均值
```

例如：

```text
开始明显下降: 约第 4 分钟
稳态性能: 峰值的 82%-86%
```
